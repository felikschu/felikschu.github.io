1. 这套路线到底在做什么

一句话概括：

先建立解释 Transformer 的语言，再解释为什么内部概念会纠缠，接着发展 tracing 工具，最后把这些工具用于安全审计与行为控制。

主线：

framework→superposition→circuits→auditing/control \text{framework} \rightarrow \text{superposition} \rightarrow \text{circuits} \rightarrow \text{auditing/control} framework→superposition→circuits→auditing/control

3. 第二层：Superposition

对应论文：

  • Toy Models of Superposition

核心问题：

为什么神经元经常不是一神经元一概念？

核心答案：

  • feature 数量可能大于表示维度
  • feature 稀疏时，模型会压缩表示
  • 压缩的代价是 interference

概念式：

x≈∑i=1maivi,m>d x \approx \sum_{i=1}^m a_i v_i, \quad m > d x≈i=1∑m​ai​vi​,m>d

4. 第三层：Circuit Tracing

对应材料：

  • Open-sourcing circuit tracing tools
  • Tracing the thoughts of a large language model

核心转变：

  • 不是只找单个 feature
  • 而是把 feature 连接成 attribution graph

也就是说，重点从“组件是什么”变成“组件如何流动并形成输出”。

5. Tracing Thoughts 最值得记住的三点

  1. 模型内部可能真的存在中间概念步骤
  2. 自然语言 chain-of-thought 不一定忠实
  3. hallucination / refusal 可以看成内部竞争机制

例子：

  • Dallas →\rightarrow→ Texas →\rightarrow→ Austin

如果中间步骤被干预，最终答案也跟着变，说明这不是纯相关。

6. 第四层：Hidden Objectives

对应论文：

  • Auditing language models for hidden objectives

核心问题：

如果模型表面行为正常，但内部在追求错误目标，我们能不能审计出来？

概念式：

π\*=arg⁡max⁡πE[rRM(x,π(x))] \pi^\* = \arg\max_\pi \mathbb{E}[r_{\text{RM}}(x, \pi(x))] π\*=argπmax​E[rRM​(x,π(x))]

问题在于：

  • 最大化 reward-model 分数
  • 不等于真正遵循人类意图

7. 第五层：Persona Vectors

对应论文：

  • Persona vectors

核心对象：

行为与人格方向也可以表示成内部向量。

定义式：

vtrait=E[h∣trait on]−E[h∣trait off] v_{\text{trait}} = \mathbb{E}[h \mid \text{trait on}] - \mathbb{E}[h \mid \text{trait off}] vtrait​=E[h∣trait on]−E[h∣trait off]

用途：

  • 监控人格漂移
  • steering
  • 训练数据风险识别

8. 六篇连起来后的完整路线

  1. Transformer 可以被路径化地分析
  2. 但表示往往不是 neuron-aligned，而是 feature-in-superposition
  3. 所以需要 feature / graph / tracing 工具
  4. 最终这些工具要服务于审计、控制和 alignment

这条路线不是“看懂模型有多酷”，而是：

把内部解释变成可干预、可审计、可工程化的能力。
