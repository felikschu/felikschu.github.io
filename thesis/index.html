Qwen3-0.6B-Base vs. Qwen3-0.6B：共享骨架下的后训练内部效应研究

这一页对应我的毕业论文主线：在控制架构变量不变的前提下，用一对共享骨架的小模型，研究后训练如何改变大语言模型内部的残差流、注意力模式和 MLP 激活空间。

1. 模型背景与定位

Qwen3 是阿里达摩院推出的第三代开源大语言模型系列，覆盖从 0.6B 到 235B 的多个参数规模，同时包含 dense 模型和混合专家模型。在这个系列里，Qwen3-0.6B 是公开可用的最小 dense 模型，参数量约 6 亿，属于非常适合做机制解释实验的“小而完整”的模型。本文选取 Qwen3-0.6B-Base 与 Qwen3-0.6B 作为对比模型对，其中 Base 是只完成预训练的原始版本，更接近一个“续写引擎”；而 Qwen3-0.6B 则是在 Base 之上进一步经过后训练得到的对话优化版本，更接近一个可直接使用的 instruct / chat 模型。

这对模型最重要的特点，不是它们规模小，而是它们共享完全相同的 Transformer 骨架。参数规模、层数、隐藏维度、注意力头数和网络拓扑都保持不变，差异主要来自后训练阶段引入的权重更新。因此，对 mechanistic interpretability 来说，这是一组非常干净的对照实验：当输入相同而内部激活不同，研究者可以更有把握地把这种变化归因到后训练，而不是归因到架构差异。

2. 共享骨架：Dense Qwen3-0.6B 架构

2.1 宏观结构

Qwen3-0.6B 采用标准的 decoder-only Transformer 架构。整个前向过程可以概括为：输入 token 先经过 embedding，进入一条宽度为 1024 的残差流；随后依次通过 28 层 Transformer block，每一层都由一个注意力子层和一个前馈子层组成；最后经过最终归一化和 language modeling head 映射为输出 logits。对本文来说，最重要的不是记住模块名称，而是明确真正贯穿全网的对象是残差流表示：注意力子层和 MLP 子层都只是在这条残差流上读取、变换、再写回自己的更新量。

2.3 单层 Transformer Block 的内部组成

单个 decoder block 可以拆成两个顺序执行的子层。首先，输入残差流经过 RMSNorm，进入注意力子层，完成 Q/K/V 投影、QK-Norm、RoPE 注入和 scaled dot-product attention，再通过输出投影写回残差流。随后，更新后的残差状态再经过一次 RMSNorm，进入 MLP 子层，使用 SwiGLU 结构进行非线性变换，最后再次写回残差流。把它写成最简化的形式，可以概括为

x → RMSNorm → Attention → 残差相加 → RMSNorm → SwiGLU MLP → 残差相加。

3. 从 Base 到 Instruct：后训练路径

Qwen3-0.6B-Base 与 Qwen3-0.6B 的差异不体现在架构，而体现在训练流程。Base 模型完成的是大规模自回归预训练，目标是从海量语料中学习语言建模能力、知识和基础推理模式；而 Qwen3-0.6B 则在此基础上继续经历后训练，一般可以概括为监督微调（SFT）与基于人类偏好的对齐阶段。用最简化的话说，Base 更像“给定前缀继续写下去”的模型，而后训练后的版本更像“理解指令并给出合适回复”的模型。

因此，两者的区别可以写得非常直接：Qwen3-0.6B-Base 是预训练收敛后的原始权重；Qwen3-0.6B 是在同一底座之上经过后训练得到的新权重。参数矩阵的形状没有变，block 的堆叠顺序没有变，注意力头数和隐藏维度没有变，真正变化的是权重值以及这些权重所诱导出的内部计算路径。

4. 为什么这对小模型适合做机制解释研究

从 mechanistic interpretability 的角度看，这对模型的价值在于变量非常干净。两者共享完全相同的 Transformer 骨架，因此任何在残差流轨迹、注意力模式、MLP 激活分布上的差异，都可以更直接地理解为后训练效应，而不是架构差异。对于解释研究来说，这种“骨架不变、训练阶段变化”的条件非常难得，因为它让研究问题从“哪个模型更强”收缩成了“同一骨架内部的计算被怎样重排了”。

同时，0.6B 这一规模也带来了实验上的现实优势。它足够小，可以完整加载、逐层导出激活、反复运行 activation patching，并且允许对多个任务组做 bootstrap / permutation 检验；但它又不是一个纯粹的玩具模型，仍然保留了真正的 LLM 结构和相当完整的行为差异。因此，小模型不是这里的妥协项，而是把研究从概念论证推进到可执行实验的关键条件。

5. 这项 thesis 的核心问题

在这个模型对上，本文真正要问的不是“Qwen3-0.6B 和 Base 哪个更强”，而是“后训练把内部计算改到了哪里”。更具体地说，本文关心的是：后训练带来的差异主要集中在早层、中层还是后层；这些差异更像是输出风格和行为路由的变化，还是更深层的语义表示与推理路径重分布；以及这种变化能否在残差流层级上被稳定观测、被 patching 操作因果验证，并在后续用 SAE 或更细粒度工具进一步展开。

因此，这一页可以收束成一个简洁结论：Qwen3-0.6B-Base 与 Qwen3-0.6B 构成了一组非常适合研究后训练内部效应的小模型对，因为它们共享相同的骨架，而差异主要来自后训练阶段。这使“后训练如何改变模型内部计算”成为一个比“哪个模型更强”更精确、也更有解释力的研究问题。
