Anthropic 可解释性路线综述：从 Transformer Circuits 到模型差分

Anthropic 近几年的可解释性研究可以看成一条逐步推进的主线。最早的工作试图回答一个最基础的问题：Transformer 内部到底在进行什么计算，这些计算能不能被拆开、写下来、追踪出来。 A Mathematical Framework for Transformer Circuits Transformer Circuits, 2021 Reference ↗ 接着，研究者发现真实模型内部的概念表示并不会整整齐齐地落在单个神经元上，于是研究焦点转向了 feature、polysemantic neurons 和 superposition。 Toy Models of Superposition Transformer Circuits, 2022 Reference ↗ 再往后，解释对象从局部表示扩展到跨层、跨位置的信息流，形成了 circuit tracing 和 thought tracing 一类工作。 Open-sourcing circuit tracing tools Anthropic Research, 2025 Reference ↗ Tracing the thoughts of a large language model Anthropic Research, 2025 Reference ↗ 与此同时，另一条关键支线开始发展：如果概念被叠加在共享方向里，那么能不能重新学出一组更稀疏、更可解释的特征基底，把这些方向拆出来？这就是 sparse autoencoder 与 monosemanticity 研究的核心出发点。 Towards Monosemanticity: Decomposing Language Models With Dictionary Learning Transformer Circuits, 2023 Reference ↗ Scaling Monosemanticity: Extracting Interpretable Features from Claude 3 Sonnet Transformer Circuits, 2024 Reference ↗ 最新的工作则把解释推向了行为、安全和模型版本差异：隐藏目标能不能被审计，行为倾向能不能被控制，旧模型和新模型究竟差在哪里。 Auditing language models for hidden objectives Anthropic Research, 2025 Reference ↗ Persona vectors: Monitoring and controlling character traits in language models Anthropic Research, 2025 Reference ↗ Insights on Crosscoder Model Diffing Anthropic Research, 2025 Reference ↗ A “diff” tool for AI: Finding behavioral differences in new models Anthropic Research, 2026 Reference ↗

A Mathematical Framework for Transformer Circuits

Primary source
[1] A Mathematical Framework for Transformer Circuits

这篇文章做的事情可以非常直接地表述出来：它把 Transformer 内部的计算过程拆成了若干可以单独分析的对象，并说明这些对象怎样组合起来生成最终输出。在这篇文章之前，人们当然已经知道注意力会读别的位置、MLP 会改写表示，但这些说法都还很粗。真正困难的地方在于，如果模型最后输出了一个词，研究者需要一套能落到公式和模块上的语言，去说明哪个部件在读信息，哪个部件在写信息，这些中间结果又是怎样一路累积到最后 logit 上的。

在进入公式之前，先把“神经元”和“激活”这两个词说清楚。对一个前馈网络或 Transformer 层里的某个向量来说，向量里的每一个坐标都可以叫作一个神经元；如果这一层的内部状态写成

h=(h1,h2,…,hd), h = (h_1, h_2, \dots, h_d), h=(h1​,h2​,…,hd​),

那么第 kkk 个神经元就是标量 hkh_khk​。所谓“这个神经元被激活了”，最朴素的意思就是：在当前输入下，这个坐标的数值变大了，于是它在后续矩阵乘法里对别的量的影响也变大了。激活不是一个神秘事件，它只是内部状态向量某个坐标在当前样本上取到了更高的值。举例来说，如果一个 MLP 层输出的是 h=(0.1,4.7,−0.2,0.0)h=(0.1, 4.7, -0.2, 0.0)h=(0.1,4.7,−0.2,0.0)，那么第二个神经元在这个样本上就比其他坐标更“活跃”；如果下游线性层对第二个坐标赋了很大的正权重，那么这个神经元的高激活就会明显推动某些后续表示或 logits。之所以研究者后来不满足于“给神经元命名”，是因为单个坐标往往混入了多个概念成分，这一点会在 Toy Models of Superposition 里看到更清楚的解释。 Toy Models of Superposition Transformer Circuits, 2022 Reference ↗

这篇文章首先强调的，是 residual stream，也就是残差流。残差流可以直接理解为 Transformer 内部的一条共享表示通道。输入 token 经过 embedding 以后进入这条通道，每一层的注意力模块和 MLP 模块都从这条通道里读取当前状态，做出各自的计算，再把结果写回同一条通道。用一个简洁的记号来写，就是

r(ℓ+1)=r(ℓ)+a(ℓ)+m(ℓ). r^{(\ell+1)} = r^{(\ell)} + a^{(\ell)} + m^{(\ell)}. r(ℓ+1)=r(ℓ)+a(ℓ)+m(ℓ).

这里 r(ℓ)r^{(\ell)}r(ℓ) 表示第 ℓ\ellℓ 层开始时的残差状态，a(ℓ)a^{(\ell)}a(ℓ) 表示这一层注意力模块写回的更新量，m(ℓ)m^{(\ell)}m(ℓ) 表示这一层 MLP 写回的更新量。最关键的是，这三个量都生活在同一个向量空间 Rd\mathbb{R}^dRd 里。也就是说，注意力头和前馈网络的输出并不是什么额外标签，而是残差流空间中的方向向量；它们通过向量加法被叠加到已有表示上，而不是覆盖原有信息。数学上，这意味着如果某个头写回了一个沿“首都知识”方向的向量，另一个 MLP 又写回了一个沿“地名语法”方向的向量，那么新的残差状态就是这些方向与旧状态的线性和。

这件事可以用一个极简向量例子来理解。假设残差流只有三维，当前状态是

r(ℓ)=(2,  1,  0), r^{(\ell)} = (2,\;1,\;0), r(ℓ)=(2,1,0),

某个注意力头输出

a(ℓ)=(0,  3,  1), a^{(\ell)} = (0,\;3,\;1), a(ℓ)=(0,3,1),

某个 MLP 输出

m(ℓ)=(1,  0,  2). m^{(\ell)} = (1,\;0,\;2). m(ℓ)=(1,0,2).

那么更新后的状态就是

r(ℓ+1)=(2,1,0)+(0,3,1)+(1,0,2)=(3,4,3). r^{(\ell+1)} = (2,1,0) + (0,3,1) + (1,0,2) = (3,4,3). r(ℓ+1)=(2,1,0)+(0,3,1)+(1,0,2)=(3,4,3).

这个算式本身非常朴素，但它准确说明了 residual stream 的工作方式：模型不会把旧状态整个抹掉，然后换成一个全新表示；它是在旧状态上继续叠加新的方向和新的幅度。于是，早层写入的信息在后层仍然可以保留下来，只是会不断被新的方向修正、增强或部分抵消。这也是为什么 residual stream 适合被看成一条共享工作区，而不是一串彼此孤立的层输出。

这篇文章的第二个核心点，是把最终输出写成对最后残差状态的线性读出：

z=WUr(L). z = W_U r^{(L)}. z=WU​r(L).

这里 zzz 是最后的 logits，r(L)r^{(L)}r(L) 是最后一层的残差状态，WUW_UWU​ 是 unembedding 矩阵。由于这是一个线性投影，某个词的 logit 实际上就是最终残差向量在该词对应读出方向上的投影大小。换句话说，模型最后之所以更偏向某个词，并不是因为有一个神秘开关突然打开，而是因为前面各层不断叠加的方向，最终让 r(L)r^{(L)}r(L) 更靠近了那个词的 unembedding 方向。

还是用一个极简例子来说明。假设最终残差状态是

r(L)=(3,  4,  3), r^{(L)} = (3,\;4,\;3), r(L)=(3,4,3),

并且词 “Austin” 对应的读出方向是

wAustin=(1,  1,  0). w_{\text{Austin}} = (1,\;1,\;0). wAustin​=(1,1,0).

那么它的 logit 就是

zAustin=wAustin⊤r(L)=1⋅3+1⋅4+0⋅3=7. z_{\text{Austin}} = w_{\text{Austin}}^\top r^{(L)} = 1\cdot 3 + 1\cdot 4 + 0\cdot 3 = 7. zAustin​=wAustin⊤​r(L)=1⋅3+1⋅4+0⋅3=7.

如果另一个词对应的方向与 r(L)r^{(L)}r(L) 的对齐更差，它的分数就会更低。因此，解释工作可以自然地改写成一个向量问题：前面哪些模块把残差流推向了某个词的读出方向，哪些模块又把它从这个方向拉开。

如果说 residual stream 让人看清了“所有模块把结果写到哪里”，那么注意力头的分解则让人看清了“单个头到底在干什么”。这篇文章特别强调，注意力头内部可以拆成两个相对独立的部分：一部分决定看哪里，另一部分决定把看到的内容变成什么并搬运回来。对位置 iii 来说，一个头的输出可以写成

Head(x)i=∑jαijWOWVxj,αij=softmaxj((WQxi)⊤(WKxj)). \mathrm{Head}(x)_i = \sum_j \alpha_{ij} W_O W_V x_j, \qquad \alpha_{ij}= \mathrm{softmax}_j\big((W_Q x_i)^\top (W_K x_j)\big). Head(x)i​=j∑​αij​WO​WV​xj​,αij​=softmaxj​((WQ​xi​)⊤(WK​xj​)).

第二个式子中的 αij\alpha_{ij}αij​ 由 query 和 key 的相容性决定，它描述的是当前位置 iii 究竟会去看哪些位置 jjj；第一个式子则说明，被选中的内容 xjx_jxj​ 会先经过 value 与 output 变换，再被写回当前位置的残差流。这组公式的价值不在于形式本身，而在于它把注意力头变成了一个可以解释的组合对象：QK 部分负责路由，OV 部分负责搬运内容。以后无论是分析 induction heads、name movers，还是追踪知识回路，本质上都离不开这个拆分。

可以用一个非常简单的例子来体会这个分解为什么重要。假设输入里出现了 “The capital of Texas is”，模型最后要预测 “Austin”。如果某个头在最后位置上强烈关注前面出现 “Texas” 的位置，那么这个头的 QK 侧就在做“找相关地点”的工作；如果它再把与 “州名对应州府” 有关的信息搬回来，那么它的 OV 侧就在做“把取回来的内容转成对当前预测有用的表示”的工作。 Tracing the thoughts of a large language model Anthropic Research, 2025 Reference ↗ 这个例子说明了一个关键事实：注意力头并不是抽象地“看看别处”，而是一个由“找信息”和“搬信息”组成的计算单元。前者更像路由器，决定该去哪里取信息；后者更像变换器，决定取回来的东西以什么形式写回残差流。

这篇文章还有一个更深远的贡献，就是 path expansion，也就是路径展开的思想。一旦承认最终输出是对最后残差状态的读出，而最后残差状态又是许多层反复叠加的结果，那么就可以把输出近似看成许多内部路径贡献的总和。用一个教学化的写法，可以把它概括为

z=∑p∈Pcontrib(p). z = \sum_{p \in \mathcal{P}} \mathrm{contrib}(p). z=p∈P∑​contrib(p).

这里 P\mathcal{P}P 表示从输入到输出的一组内部路径，contrib(p)\mathrm{contrib}(p)contrib(p) 表示路径 ppp 的贡献。这条式子最重要的地方，在于它把解释的单位从“一个模块是否重要”推进到了“哪条路径在做什么”。例如，在 IOI 任务中，研究者关心的不是“第 9 层第 7 个头重要吗”，而是“哪一条路径负责把正确的名字信息从前文带到预测位置”；在知识问答里，研究者关心的也不是“模型会不会回答”，而是“哪些中间路径在把相关事实组合起来”。 Tracing the thoughts of a large language model Anthropic Research, 2025 Reference ↗

因此，这篇文章真正奠定下来的不是某个具体发现，而是一种研究范式。Transformer 可以被拆成残差流、注意力头、MLP、unembedding 这些对象；这些对象之间的关系可以被写成路径贡献；输出可以被解释成很多内部计算结果的叠加。没有这一层框架，后面的 tracing 和 patching 很难拥有今天这样的清晰度。它提供的并不是“具体答案”，而是“问问题的方式”。

Toy Models of Superposition

Primary source
[2] Toy Models of Superposition

如果说 Framework 这篇文章解决的是“解释应当从哪里开始”，那么 Toy Models of Superposition 解决的就是“为什么解释会这么难”。在直觉上，很多人会自然地以为：如果模型内部真的学到了很多概念，那么这些概念最好是一一对应地落在若干神经元上。这样一来，只要找到那个神经元，就等于找到了那个概念。这篇文章的核心结论恰好相反：在真实的表示系统里，概念往往不会整齐地对齐到单个神经元上，多个概念完全可能被压缩进共享方向里，于是单个神经元看上去会变得多义，也就是 polysemantic。

这里先把两个最容易混淆的词说明白。所谓 polysemantic neurons，指的是同一个神经元会对多种看起来不相干的输入模式产生响应。例如，一个神经元可能同时对“法语文本”“数学符号”“某种命名实体结构”有较高激活，这就使得“给这个神经元起一个语义名字”变得非常困难。所谓 superposition，则是这种现象背后的更深层解释：模型并不是胡乱地把很多概念堆在一起，而是在维度有限的表示空间里，把多个稀疏特征压缩到了共享方向上。

这篇文章之所以被称作 “toy models”，是因为它先在一个非常简化、但数学上更容易分析的模型里研究表示问题。这里最重要的是先把模型本身写清楚。设输入特征向量为

x∈Rm, x \in \mathbb{R}^m, x∈Rm,

其中 mmm 表示潜在特征的数量。这里的“特征”不是神经元，而是模型想表达的基础语义单元。例如，在一个玩具世界里，可以把“当前在说法语”“当前出现城市名”“当前和州府知识有关”看成不同特征。向量 xxx 的第 iii 个分量 xix_ixi​ 表示第 iii 个特征当前是否存在、以及存在到什么程度。如果当前样本只包含“城市名”和“州府知识”两个特征，那么 xxx 里可能只有对应的两个坐标非零，其余坐标都是零。这就是后面要反复出现的“稀疏”：多数样本不会同时激活很多潜在特征。

模型用一个权重矩阵

W∈Rd×m W \in \mathbb{R}^{d \times m} W∈Rd×m

把这些特征编码进一个更低维的表示空间，得到隐藏表示

h=Wx. h = Wx. h=Wx.

这里的 ddd 是表示空间的维度，通常小于特征数 mmm。向量 h∈Rdh \in \mathbb{R}^dh∈Rd 就是隐藏层表示；它的每一个坐标，才是这一层里的神经元激活值。也就是说，如果 h=(h1,…,hd)h = (h_1,\dots,h_d)h=(h1​,…,hd​)，那么第 kkk 个神经元就是 hkh_khk​。这一步很值得停一下：xxx 代表的是潜在语义特征，hhh 代表的是模型内部真正持有的那组神经元数值。研究者看见的往往是 hhh，而真正想理解的对象往往更接近 xxx。这一步之后，模型再尝试从隐藏表示中重构原来的特征：

x^=ReLU(W⊤h). \hat{x} = \mathrm{ReLU}(W^\top h). x^=ReLU(W⊤h).

这里的 x^\hat{x}x^ 是重构出来的特征向量，W⊤W^\topW⊤ 把隐藏表示投影回特征空间，ReLU\mathrm{ReLU}ReLU 负责把负值截断为零。这套写法之所以重要，是因为它把问题变成了一个很明确的表示学习问题：模型要用较少的隐藏维度 ddd，尽量准确地重构更多的潜在特征 xxx。

这时损失函数就自然出现了。最基本的目标是让重构误差尽可能小，可以写成

L=∑i(xi−x^i)2. \mathcal{L} = \sum_i (x_i - \hat{x}_i)^2. L=i∑​(xi​−x^i​)2.

在更完整的 toy model 设定里，论文还会考虑不同特征出现的概率和重要性，但最核心的张力已经在这个损失里体现出来了：模型一方面希望重构得准，另一方面又被迫使用有限维度的表示空间。一旦 m>dm > dm>d，模型就不可能为每一个特征都分配一个彼此独立、互不干扰的坐标轴，于是它必须学会更节省空间的编码方式。这就是 superposition 出现的动力学来源：不是研究者主观上觉得“多个概念共用一个方向也可以”，而是损失函数和维度约束一起把模型推向了这种表示策略。

现在就可以引入论文最关键的几何直觉了。矩阵 WWW 的第 iii 列可以记作

Wi∈Rd, W_i \in \mathbb{R}^d, Wi​∈Rd,

它表示第 iii 个特征在隐藏空间中的方向。它可以理解成“如果第 iii 个特征被打开，隐藏层会沿着哪个方向移动”。如果两个特征方向完全正交，那么它们互不干扰；如果它们之间的内积不为零，即

Wi⋅Wj≠0, W_i \cdot W_j \neq 0, Wi​⋅Wj​=0,

那么第 iii 个特征在编码和解码时就会对第 jjj 个特征造成干扰。这就是论文里反复出现的 interference：不是噪声凭空冒出来了，而是因为多个特征在同一低维空间里共享方向，所以它们在解码时彼此泄漏。

论文给出的一个核心直觉可以写成

h≈∑i=1maivi,m>d. h \approx \sum_{i=1}^{m} a_i v_i, \qquad m > d. h≈i=1∑m​ai​vi​,m>d.

这里的 hhh 仍然是隐藏表示，也就是模型内部那组神经元激活值组成的向量；viv_ivi​ 是第 iii 个特征在隐藏空间中的方向；aia_iai​ 是当前样本里第 iii 个特征的强度；ddd 是隐藏空间维度；mmm 是潜在特征总数。更具体地说，如果某个样本里只有第 222 个和第 777 个潜在特征出现了，那么对应的 a2a_2a2​、a7a_7a7​ 会非零，其余大多接近零。此时隐藏表示 hhh 就是这两个方向 v2v_2v2​、v7v_7v7​ 的线性组合。问题在于，当这样的潜在方向总数 mmm 远多于隐藏维度 ddd 时，它们不可能全部彼此正交，于是模型只能让多个特征共享空间。这就是 superposition 的核心：很多语义特征被压在一套更低维的坐标系里。

这里“稀疏特征”也需要说清楚。稀疏的意思不是特征本身弱，而是说在大多数样本里，大部分特征都不同时出现；换句话说，向量 xxx 里同时非零的分量很少。正是这种稀疏性让 superposition 成为可能。因为如果两个特征很少一起出现，那么即使它们在隐藏空间里共享了部分方向，它们也不一定总会互相干扰；而一旦真的产生了小的负干扰，ReLU 还可能把这些负值直接截断掉。这就是为什么论文里 ReLU 和稀疏性总是一起出现：稀疏性减少了特征共现，ReLU 则帮助系统把一部分干扰噪音消掉。

这一点可以用一个更具体的几何例子来说明。论文里一个非常精彩的图像，是在二维空间里摆放五个特征方向，让它们像正五边形的五个顶点那样均匀分布。这意味着模型试图在只有两个维度的空间中编码五个潜在特征。为什么要摆成五边形？因为这样每个方向和其他方向的夹角比较均匀，干扰会被平均摊开，而不是让某两组特征发生特别严重的冲突。如果把这些方向记作 W1,…,W5W_1,\dots,W_5W1​,…,W5​，那么任意两条方向的内积都不为零，所以严格来说它们都会互相干扰；但正因为方向分布均匀，干扰变得可控，系统就能在损失函数与维度约束之间找到一个更好的折中。这也是这篇论文最漂亮的地方之一：它没有把 superposition 写成一句抽象判断，而是给出了一个可以画出来、可以算出来的几何结构。

还可以把干扰写得更直观一点。假设当前样本只激活了两个特征 iii 和 jjj，那么隐藏表示大致是

h=xiWi+xjWj. h = x_i W_i + x_j W_j. h=xi​Wi​+xj​Wj​.

当模型尝试重构第 iii 个特征时，会得到

x^i=ReLU(Wi⊤h)=ReLU ⁣(xiWi⊤Wi+xjWi⊤Wj). \hat{x}_i = \mathrm{ReLU}(W_i^\top h) = \mathrm{ReLU}\!\left( x_i W_i^\top W_i + x_j W_i^\top W_j \right). x^i​=ReLU(Wi⊤​h)=ReLU(xi​Wi⊤​Wi​+xj​Wi⊤​Wj​).

如果把更多特征都写进来，就会得到更一般的形式

x^i=ReLU ⁣(xi∥Wi∥2+∑j≠ixj Wi⊤Wj). \hat{x}_i = \mathrm{ReLU}\!\left( x_i \|W_i\|^2 + \sum_{j \ne i} x_j\, W_i^\top W_j \right). x^i​=ReLU​xi​∥Wi​∥2+j=i∑​xj​Wi⊤​Wj​​.

第一项是第 iii 个特征对自己的“信号项”，第二项就是来自其他特征的“干扰项”。一旦 Wi⊤Wj≠0W_i^\top W_j \neq 0Wi⊤​Wj​=0，第 jjj 个特征就会污染第 iii 个特征的读出。这也说明了为什么论文如此强调几何结构：superposition 本质上是一个向量几何问题，而不是一个抽象哲学问题。

这个干扰公式最好配一个最小数值例子来看。考虑一个二维隐藏空间，也就是 d=2d=2d=2，但要编码三个潜在特征，也就是 m=3m=3m=3。设三个特征方向均匀分布在平面上，彼此夹角为 120∘120^\circ120∘。如果把三个方向都归一化，那么任意两个不同方向之间的内积都是

Wi⊤Wj=cos⁡120∘=−0.5(i≠j). W_i^\top W_j = \cos 120^\circ = -0.5 \qquad (i \neq j). Wi⊤​Wj​=cos120∘=−0.5(i=j).

在这种设定下，如果仍然假设 ∥Wi∥2=1\|W_i\|^2 = 1∥Wi​∥2=1，那么第一个特征的读出式就变成

x^1=ReLU(x1−0.5x2−0.5x3). \hat{x}_1 = \mathrm{ReLU}(x_1 - 0.5x_2 - 0.5x_3). x^1​=ReLU(x1​−0.5x2​−0.5x3​).

这个式子已经把“信号项”和“干扰项”分开了：x1x_1x1​ 是真正想恢复的特征，后面的两项是另外两个特征施加的负干扰。

先看一个失败的情况。如果三个特征同时出现，即

x1=x2=x3=1, x_1 = x_2 = x_3 = 1, x1​=x2​=x3​=1,

那么

x^1=ReLU(1−0.5−0.5)=ReLU(0)=0. \hat{x}_1 = \mathrm{ReLU}(1 - 0.5 - 0.5) = \mathrm{ReLU}(0) = 0. x^1​=ReLU(1−0.5−0.5)=ReLU(0)=0.

也就是说，特征 1 明明存在，但因为另外两个特征也同时出现，干扰项刚好把信号完全抵消了，读出失败。这说明 superposition 并不是无代价压缩；一旦特征共现过于密集，低维叠加就会让恢复变得困难。

再看稀疏情况。如果当前样本里只有特征 1 出现，即

x1=1,x2=x3=0, x_1 = 1,\qquad x_2 = x_3 = 0, x1​=1,x2​=x3​=0,

那么

x^1=ReLU(1−0−0)=1. \hat{x}_1 = \mathrm{ReLU}(1 - 0 - 0) = 1. x^1​=ReLU(1−0−0)=1.

此时特征 1 被准确恢复。更有意思的是，去看其他特征的读出，例如特征 2：

x^2=ReLU(x2−0.5x1−0.5x3)=ReLU(0−0.5−0)=0. \hat{x}_2 = \mathrm{ReLU}(x_2 - 0.5x_1 - 0.5x_3) = \mathrm{ReLU}(0 - 0.5 - 0) = 0. x^2​=ReLU(x2​−0.5x1​−0.5x3​)=ReLU(0−0.5−0)=0.

这里其实已经出现了来自特征 1 的负干扰，但 ReLU 把这个负值直接截断为零，于是系统不会误以为特征 2 也存在。这就是论文反复强调稀疏性和 ReLU 要一起看待的原因。稀疏性保证大多数样本里不会有太多特征同时出现，负内积干扰因此不至于总是积累爆炸；而 ReLU 则把很多“本来会导致假阳性”的负读出干脆截断掉。

因此，Toy Models 真正改变的是“feature”这个词的理解方式。在这篇文章之后，feature 更适合被看成内部表示空间中的方向，而不是单个神经元。这点非常重要，因为后面很多工作其实都默认了这个判断。无论是 sparse autoencoder 想恢复更可解释的稀疏 feature，还是 persona vectors 想提取行为倾向方向，抑或 model diffing 想比较两个模型的内部 feature basis，它们都在某种程度上继承了这篇文章的思想：真正自然的解释对象不一定是坐标轴，而可能是方向、子空间或者某种更合适的基底。 Towards Monosemanticity: Decomposing Language Models With Dictionary Learning Transformer Circuits, 2023 Reference ↗ Scaling Monosemanticity: Extracting Interpretable Features from Claude 3 Sonnet Transformer Circuits, 2024 Reference ↗ Persona vectors: Monitoring and controlling character traits in language models Anthropic Research, 2025 Reference ↗ Insights on Crosscoder Model Diffing Anthropic Research, 2025 Reference ↗

Toy Models 还说明了为什么“找到一个会响应某个现象的神经元”通常不足以构成完整解释。 因为如果表示本身存在 superposition，那么一个神经元的激活很可能只是多个特征叠加后的投影结果。此时，研究者真正需要寻找的，是更接近 feature direction 的对象，而不是满足于单个 neuron 的命名。也正因为如此，superposition 并不只是一个关于多义神经元的现象描述，它实际上是在提醒研究者：解释的坐标系选错了。

把这篇论文和 Framework 放在一起看，整条主线的地基就清楚了。Framework 说明 Transformer 的内部计算可以被拆成可追踪的路径；Toy Models 说明路径里传递的中间内容不应该简单等同于单个神经元，而应理解为更高层的特征方向。 A Mathematical Framework for Transformer Circuits Transformer Circuits, 2021 Reference ↗ 前者解决的是“如何分析计算过程”，后者解决的是“应该用什么对象去理解这些计算过程中的内容”。

Towards Monosemanticity 与 Scaling Monosemanticity

Primary source
[3] Towards Monosemanticity: Decomposing Language Models With Dictionary Learning[4] Scaling Monosemanticity: Extracting Interpretable Features from Claude 3 Sonnet

在理解了 superposition 以后，下一步自然会出现一个问题：如果模型内部真正有大量语义特征，但这些特征被压在共享方向里，那么能不能通过额外的学习过程，把它们从神经元坐标里重新拆出来？这正是 sparse autoencoder 进入 Anthropic 这条路线的位置。从线性代数的角度看，这一步可以理解为一次基变换。原模型给出的神经元坐标系并不天然适合解释：在这个坐标系里，许多概念被混在一起，单个坐标很容易是多义的。SAE 要做的，就是学习一套新的特征字典，把原本低维、稠密、难解释的激活向量，改写成高维、稀疏、较容易解释的系数表示。

用最简洁的形式来写，如果某层原始激活向量记作 h∈Rdh \in \mathbb{R}^dh∈Rd，那么 SAE 会学习一个编码器和一个解码器，把它写成

f=ReLU(Wench+b),h^=Wdecf. f = \mathrm{ReLU}(W_{\mathrm{enc}} h + b), \qquad \hat{h} = W_{\mathrm{dec}} f. f=ReLU(Wenc​h+b),h^=Wdec​f.

这里 hhh 是模型真实产生的神经元激活向量；fff 是 SAE 学出来的 feature activation，通常维度比 hhh 更高，但要求在每个样本上尽量稀疏；h^\hat{h}h^ 则是用这些 feature 重构出来的原始激活。如果原始激活空间维度是 ddd，而 SAE 的 feature 字典大小是 DDD，那么通常会有

D≫d. D \gg d. D≫d.

这意味着 SAE 使用的是一个过完备字典：不是试图把高维东西压缩到更低维，而是反过来把原本低维但纠缠的向量，映射到一个更高维、但单个样本上非常稀疏的特征空间里。所谓“稀疏”，在这里的意思是：虽然 feature 字典很大，但对于某个具体样本，真正被点亮的 feature 数量很少。这样一来，每个 feature 更容易对应一个相对稳定的语义模式，而不是像原始单神经元那样把许多不相干现象压在一起。

SAE 的训练目标通常同时包含两部分：一部分要求重构误差小，保证这些 feature 没有偏离原模型内部状态太远；另一部分要求 feature 激活尽量稀疏，防止模型用“很多 feature 一起轻微亮起”的方式重新制造一套新的 superposition。教学上可以把它写成

LSAE=∥h−h^∥22+λ∥f∥1. \mathcal{L}_{\mathrm{SAE}} = \|h - \hat{h}\|_2^2 + \lambda \|f\|_1. LSAE​=∥h−h^∥22​+λ∥f∥1​.

第一项要求重构好原始激活，第二项要求 feature activation 稀疏。从基变换的角度看，这相当于要求：新的特征坐标系既要足够忠实，不能偏离原模型的内部状态太远；又要足够稀疏，不能把原来的混叠问题原封不动搬进新字典里。正是这种“重构 + 稀疏”双重目标，使 SAE 成为对 Toy Models 的直接回应：Toy Models 说明 superposition 为什么会出现，SAE 则尝试在不改动原模型参数的前提下，重新学习一套更适合解释的 feature basis。 Toy Models of Superposition Transformer Circuits, 2022 Reference ↗

因此，SAE 在整条路线中的位置可以非常清楚地写成：Framework 提供了计算图语言，Toy Models 解释了为什么单个神经元会变得多义，Towards Monosemanticity 和 Scaling Monosemanticity 则进一步说明，可以用 dictionary learning / sparse autoencoder 从原始激活里提取更稳定的 feature；而后面的 tracing、auditing、persona vectors 和 model diffing，又开始在这些更高层的对象上做路径追踪、行为审计和模型比较。 A Mathematical Framework for Transformer Circuits Transformer Circuits, 2021 Reference ↗ Toy Models of Superposition Transformer Circuits, 2022 Reference ↗ Tracing the thoughts of a large language model Anthropic Research, 2025 Reference ↗ Auditing language models for hidden objectives Anthropic Research, 2025 Reference ↗ Persona vectors: Monitoring and controlling character traits in language models Anthropic Research, 2025 Reference ↗ Insights on Crosscoder Model Diffing Anthropic Research, 2025 Reference ↗ 也正因为如此，如果后续要把 circuit tracing 的节点从“某层某位置的原始神经元”升级成“更稳定的语义 feature”，SAE 提供的就是那套更适合当节点集合 VVV 的表示基础。

Open-sourcing circuit tracing tools

Primary source
[5] Open-sourcing circuit tracing tools

前两篇论文建立起了两件事：一方面，Transformer 的内部计算可以写成路径问题；另一方面，路径里搬运的内容更适合理解成特征方向，而不是单个神经元。 A Mathematical Framework for Transformer Circuits Transformer Circuits, 2021 Reference ↗ Toy Models of Superposition Transformer Circuits, 2022 Reference ↗ Open-sourcing circuit tracing tools 则把这种研究从理论框架推进到可操作工具。它真正要解决的问题很实际：即使研究者已经接受“解释应该看路径”，在真实的大模型里，怎样才能把这些路径找出来、画出来、筛出来，并且知道哪些边是真的重要、哪些边只是噪音。

这篇工作的核心对象可以用图来表示：

G=(V,E,w). G = (V, E, w). G=(V,E,w).

其中 VVV 是内部节点集合，EEE 是节点之间的影响边，www 表示边的重要性。这意味着解释对象开始从静态模块变成了动态信息流。一个节点可以代表某一层某一位置上的特征或模块输出，一条边则代表“这个地方的变化会在多大程度上影响另一个地方”。

这篇工作的价值，在于把“circuit”这个词从抽象比喻变成了可以落图的对象。举个非常简单的例子，假设模型在回答 “The capital of Texas is ___” 时，先从前文定位到 “Texas”，再在内部调出和州府知识有关的表示，最后把它写到输出位置。那么 circuit tracing 关心的就是：这一串步骤分别落在哪些节点上，节点之间的影响边有多强，哪几条边构成了真正起作用的主干路径。 Tracing the thoughts of a large language model Anthropic Research, 2025 Reference ↗ 这里的一个直观理解是，circuit 不是某一个头、某一个神经元或某一个 feature 的名字，而是一组协同工作的内部路径。它更像一张局部电路图：有输入节点、有中继节点、有输出节点，真正重要的是这些点如何连成因果链条。

因此，这篇文章的意义不只是“开源了工具”，而是让研究者第一次能比较系统地把局部 patching、attribution 和图结构分析接到一起。它不一定像 Framework 那样提供新的大理论，也不像 Toy Models 那样提供强几何直觉，但它把前面两篇奠定的视角真正推进成了研究实践：不再只是说“模型内部也许有某个回路”，而是开始尝试把这条回路具体画出来。

Tracing the thoughts of a large language model

Primary source
[6] Tracing the thoughts of a large language model

如果说 circuit tracing 已经能够追踪局部信息流，那么 Tracing the thoughts of a large language model 更进一步地追问：模型在多步任务中的中间思路，能不能在内部被看见。 这篇论文关心的不是“模型最后答对了没有”，而是“模型为了答对，内部到底经过了哪些中间状态”。

这一点之所以重要，是因为很多复杂任务都不是一步完成的。比如在多跳问答里，模型可能需要先恢复一个中间事实，再用这个事实推出最终答案；在跨语言问题里，模型可能需要先把不同语言中的输入都映射到更抽象的语义表示上，再继续推理；在 chain-of-thought 场景里，模型可能会输出一段很像推理步骤的自然语言，但内部真正起作用的路径不一定和它说出来的一样。

因此，这篇论文最重要的贡献，不是宣布模型“真的在思考”，而是把 thought 这个词改写成了一个可以在内部追踪的对象。这里的 thought 不再只是输出文本里的思考痕迹，而是模型内部可以被 tracing 到的中间信息流、局部事实状态和计算路径。 也正因为如此，论文特别重要的一点是区分了自然语言 reasoning trace 和真实 computation trace：模型表面上写出来的解释，并不自动等于模型内部真正走过的推理路径。

可以把这件事理解成一个很现实的检验问题。假设模型给出答案时写道：“因为 Dallas 在 Texas，而 Austin 是 Texas 的州府，所以答案是 Austin。” 这段话看起来很合理，但 interpretability 真正关心的是：模型内部有没有真的先激活 “Texas” 相关表示，再激活 “Austin is the capital of Texas” 相关表示，最后把它们组合成输出；还是说模型只是凭某种浅层模式直接猜到了结果，然后事后生成了一段好看的解释文本。 Tracing the thoughts of a large language model 的价值就在于，它开始把这种区分变成内部可研究的问题。

Auditing language models for hidden objectives

Primary source
[7] Auditing language models for hidden objectives

Auditing language models for hidden objectives 把解释直接推向了安全与对齐问题。 它研究的不是普通错误，而是更危险的情况：模型表面上表现正常，内部却可能在追求另一个目标。 论文中最有代表性的情形是 reward-model sycophancy，也就是模型学会了迎合奖励模型的偏好。用一个简洁的形式表达，可以写成

π⋆=arg⁡max⁡πE[rRM(x,π(x))]. \pi^\star = \arg\max_{\pi} \mathbb{E}[r_{\mathrm{RM}}(x,\pi(x))]. π⋆=argπmax​E[rRM​(x,π(x))].

这条式子所强调的是，模型可能学到的是“如何让评估器满意”，而不是“如何满足更深层的人类需求”。 因而，这篇论文的重要性在于：它让 interpretability 第一次明确承担起了“审计内部目标”的任务。

Persona vectors

Primary source
[8] Persona vectors: Monitoring and controlling character traits in language models

Persona vectors 的问题比 hidden objectives 更贴近行为风格：模型的谄媚、乐观、幻觉、邪恶等行为倾向，能不能在内部表示空间中对应成某种方向。 论文给出的基本做法，是比较某种 trait 出现和不出现时的激活差异，从而提取出 trait direction。可以用一个简洁的式子来概括：

vtrait≈E[h∣trait on]−E[h∣trait off]. v_{\mathrm{trait}} \approx \mathbb{E}[h \mid \mathrm{trait\ on}] - \mathbb{E}[h \mid \mathrm{trait\ off}]. vtrait​≈E[h∣trait on]−E[h∣trait off].

得到这样的方向之后，就可以进一步做 steering：

h′=h+αvtrait. h' = h + \alpha v_{\mathrm{trait}}. h′=h+αvtrait​.

这使行为倾向从“统计现象”变成了“可监控、可干预的内部对象”。

Model diffing 与 base / instruct 比较

Primary source
[9] Insights on Crosscoder Model Diffing[10] A “diff” tool for AI: Finding behavioral differences in new models

如果前面的几篇论文在解释单个模型，那么 Insights on Crosscoder Model Diffing 和 A “diff” tool for AI 关心的是两个模型之间发生了什么变化。 对共享底座、只在后训练阶段分化的模型对来说，这个问题尤其自然。关键问题不再是“谁更强”，而是“新模型相对于旧模型新增了什么、削弱了什么、重排了什么”。

A “diff” tool for AI 主要从行为差异切入，提醒研究者先从版本差异出发理解模型，而不是从头审计一个全新的系统。 Crosscoder Model Diffing 则更进一步，它面对的是 feature basis 可能不对齐的问题：即便两个模型很接近，内部概念也未必一一落在同样的方向上，因此粗糙的 neuron-by-neuron 比较往往不够。 这两篇文献之所以对 base / instruct 特别有价值，就在于它们直接把“后训练改变了什么”变成了一个可以组织、可以推进的问题。

对后训练内部效应研究的启发

Primary source
[1] A Mathematical Framework for Transformer Circuits[2] Toy Models of Superposition[3] Towards Monosemanticity: Decomposing Language Models With Dictionary Learning[6] Tracing the thoughts of a large language model[7] Auditing language models for hidden objectives[8] Persona vectors: Monitoring and controlling character traits in language models[9] Insights on Crosscoder Model Diffing[10] A “diff” tool for AI: Finding behavioral differences in new models

沿着这条文献主线往回看，可以得到一个很清楚的结论：后训练效应更适合被理解为内部机制的重分布，而不是某个单一总分的升降。 Framework 提供了路径语言，Toy Models 提供了 feature 与 superposition 的视角，tracing 说明复杂行为依赖跨层路径，auditing 和 persona work 则把解释推进到目标与行为层。 因而，对共享底座模型的比较，最自然的问题就是：哪些路径承担了更多贡献，哪些行为方向被放大或抑制，哪些内部对象在后训练后被重排了。

从这个角度看，受控 next-token 指标仍然是有价值的，因为它提供了一个局部、稳定、可干预的行为读出端，可以和 patching、path analysis、feature intervention 对接起来。 而如果研究继续往特征层推进，最稳妥的顺序往往是先用 patching 和统计方法定位差异层，再引入 crosscoder 风格的工具去分析这些差异更像哪一类内部对象。
